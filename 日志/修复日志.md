### **技术原理解析**

#### **Q1: 为什么用相对导入（`.utils`）而不是绝对导入（`utils`）？**

**答案：**

```python
# ❌ 错误：绝对导入（只能在顶层使用）
from utils import logger

# ✅ 正确：相对导入（包内模块互相引用）
from .utils import logger
```


**面试考点：**
- **Python 包结构**：有 `__init__.py` 的目录才是包
- **相对导入语法**：`.` 表示当前包，`..` 表示父包
- **避免命名冲突**：相对导入确保导入的是包内模块

> [!IMPORTANT]
>
> 此处有误，要使用绝对导入



---

#### **Q2: 为什么需要 `ParentChunkStore` 这个类？**

**技术原理：**

```
RAG 分层检索策略：
┌─────────────────────────────────┐
│ L1 (大 chunk) │ 2000 tokens     │ ← 完整性高，适合生成
├─────────────────────────────────┤
│ L2 (中 chunk) │ 800 tokens      │ ← 平衡
├─────────────────────────────────┤
│ L3 (小 chunk) │ 300 tokens      │ ← 语义精确，适合检索
└─────────────────────────────────┘

检索流程：
1. 用户查询 → 向量化 L3
2. 检索到多个相关 L3（如 3 个）
3. Auto-merging: 如果这 3 个 L3 同属一个 L2
4. 返回 L2 而非 L3 → 上下文更完整
```


**面试考点：**
- **父子块索引（Parent-Child Indexing）**：检索子块，返回父块
- **Chunk Size 选择**：太小丢失上下文，太大引入噪声
- **Auto-merging**：动态决定返回哪个层级的块

## 🔬 **技术原理解析**

### **Q1: 为什么会有这个错误？**

**Python 的两种运行模式：**

```bash
# 模式 1：作为脚本运行
python backend/agent.py
# → Python 认为 agent.py 是独立文件
# → 不支持相对导入（.tools）

# 模式 2：作为模块运行
python -m backend.agent
# → Python 知道 agent 在 backend 包内
# → 支持相对导入（.tools）
```


**Flask 的情况：**
```python
# app.py 中导入
from agent import chat_with_agent_stream

# Flask 直接执行 agent.py（模式 1）
# 所以不能用相对导入
```

### **Q2: 相对导入 vs 绝对导入**

| 导入方式     | 语法                      | 使用场景     | 示例                        |
| ------------ | ------------------------- | ------------ | --------------------------- |
| **绝对导入** | `from module import xxx`  | 任何情况     | `from utils import logger`  |
| **相对导入** | `from .module import xxx` | 包内互相引用 | `from .utils import logger` |

**最佳实践：**

```python
# ✅ 推荐：统一使用绝对导入
# 清晰、明确、不易出错
from backend.tools import search_knowledge_base
from backend.utils import logger

# ⚠️ 慎用：相对导入
# 只在确定是包内调用时使用
from .tools import search_knowledge_base
```

### **Q3: `__init__.py` 的作用**

**技术原理：**

```
目录结构：
backend/
├── __init__.py      ← 标记这是 Python 包
├── agent.py
├── app.py
└── tools.py

没有 __init__.py:
backend/            ← 只是普通文件夹
├── agent.py        ← 不能互相导入
└── tools.py
```


**`__init__.py` 的内容：**
```python
# 最小化：空文件即可
# 或者添加包元数据
__version__ = '1.0.0'
__author__ = 'Your Name'

# 或导出常用模块
from .agent import chat_with_agent_stream
from .tools import search_knowledge_base
```


**面试考点：**
- **Python 3.3+**：支持隐式命名空间包（不需要 `__init__.py`）
- **但推荐添加**：明确标识、版本管理、初始化逻辑

### **常考问题**

**Q: Python 的导入机制是什么？**

**答案：**
```
1. sys.modules 缓存查找
2. 内置模块查找
3. sys.path 路径查找
   - 当前目录
   - PYTHONPATH 环境变量
   - 标准库
   - site-packages
4. 找到后执行模块代码
5. 缓存到 sys.modules
```


**Q: 如何解决循环导入？**

**解决方案：**
```python
# 方案 1：延迟导入（推荐）
def function_a():
    from module_b import function_b  # 用时再导入
    return function_b()

# 方案 2：重构代码结构
# 将共用代码提取到独立模块

# 方案 3：使用 importlib
import importlib
module_b = importlib.import_module('module_b')
```
