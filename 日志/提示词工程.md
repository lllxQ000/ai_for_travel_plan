# 提示词优化：从内容丰富度到 Agent Prompt 设计

在 RAG 系统中，提示词（Prompt）是影响生成质量的关键因素。你提到的两个问题——“输出内容丰富度”和“Agent Prompt 模板设计”——正是提升系统表现的核心。以下结合本项目，详细解答。

---

## 一、输出内容丰富度：通过最终生成提示词优化

是的，**你完全可以通过优化最后给 LLM 的生成提示词来提升输出的内容丰富度**。即使检索到的上下文相同，一个精心设计的提示词能让 LLM 产出更详细、更有条理、更贴合用户需求的答案。

### 1.1 关键技巧

#### a) 明确角色与任务
告诉模型“你是谁”、“要做什么”。例如：
```
你是一位资深的旅行规划师，擅长根据用户偏好和最新旅游信息，制定详细、实用的行程方案。
```

#### b) 结构化输出要求
指定输出的格式，可以让内容更易读、信息更密集。例如：
```
请按以下格式输出：
## 行程概览
- 总天数：...
- 适合人群：...

## 每日详细行程
### 第1天
- 上午：...（景点+建议停留时间）
- 中午：...（推荐餐厅+特色菜）
- 下午：...
- 晚上：...
### 第2天
...
## 摄影机位提示（如果有）
...
```

#### c) 提供示例（Few-Shot）
在提示词中加入一个简短的示例输出，能极大改善模型对格式和细节的理解。

#### d) 约束与扩展
- **约束**：明确要求“必须基于提供的上下文”、“禁止编造事实”。
- **扩展**：要求“如果上下文中有多个选择，请列出至少3个”、“详细说明每个景点的亮点”。

#### e) 动态变量注入
将用户偏好（如“预算：中等”）和检索到的上下文动态填入提示词。

### 1.2 代码实现（在 `agent.py` 的生成部分）

假设你已经在 `agent.py` 中构造了 `messages` 列表，其中包含系统提示、历史消息、用户问题以及检索到的上下文。你可以这样优化系统提示：

```python
system_prompt = (
    "你是一位专业的旅行规划助手。\n"
    "你的任务是根据用户偏好和提供的旅游信息，制定详细、实用、个性化的行程。\n"
    "请按照以下格式输出：\n"
    "## 行程概览\n"
    "- 总天数：...\n"
    "- 适合人群：...\n"
    "## 每日详细行程\n"
    "### 第1天\n"
    "- 上午：...（建议停留时间）\n"
    "- 中午：...（推荐餐厅+特色菜）\n"
    "- 下午：...\n"
    "- 晚上：...\n"
    "### 第2天\n"
    "...\n"
    "## 特别提示\n"
    "- 摄影机位：...\n"
    "- 当地美食：...\n"
    "如果提供的上下文不足以支持详细规划，请诚实说明，不要编造。"
)
```

然后将其与检索到的上下文、用户问题一起作为消息发送给 LLM。

## 二、Agent Prompt 模板的设计关键点

Agent 的 Prompt（通常是一个系统消息）决定了 Agent 如何理解用户输入、调用工具、整合结果。设计不当会导致工具滥用、逻辑混乱、重复调用等问题。

### 2.1 核心设计原则

#### 1. **明确角色与能力边界**
告诉 Agent 它是什么、能做什么、不能做什么。例如：
> “你是一个智能助手，可以查询天气（使用 get_current_weather）或从知识库检索信息（使用 search_knowledge_base）。对于其他问题，直接回答或拒绝。”

#### 2. **工具使用规则清晰**
- **何时调用工具**：列出触发条件。
- **如何调用**：说明参数格式。
- **调用后行为**：强调“收到工具结果后必须立即生成最终答案，不得再次调用工具”。
- **避免重复调用**：用指令限制“同一轮对话中，每个工具最多调用一次”。

#### 3. **上下文整合指示**
告诉 Agent 如何利用检索到的上下文：
> “当使用 search_knowledge_base 后，你会得到一段上下文。请基于此上下文回答，如果上下文不足，诚实说不知道。”

#### 4. **输出格式约束**
与最终生成提示类似，可要求 Agent 的输出符合某种结构。

#### 5. **错误处理与降级**
指示 Agent 在工具调用失败时的行为：
> “如果工具调用失败，请告知用户当前服务不可用，不要重复尝试。”

#### 6. **避免幻觉**
明确要求“不要编造事实”，尤其是在工具返回空结果时。

### 2.2 示例：优化后的 Agent 系统提示

以下是我们项目中使用的优化版本（已在 `agent.py` 中体现）：

```python
system_prompt = (
    "You are a helpful assistant that can answer questions using a knowledge base.\n"
    "When users ask about travel planning, document content, or specific information, "
    "use the search_knowledge_base tool to retrieve relevant context.\n"
    "For weather queries, use get_current_weather.\n"
    "Do not call the same tool repeatedly in one turn. At most one knowledge tool call per turn.\n"
    "After receiving search_knowledge_base result, you MUST immediately produce the Final Answer based on that result.\n"
    "If the retrieved context is insufficient, answer honestly that you don't know instead of making up facts.\n"
    "If the user asks for a detailed itinerary, organize your answer with headings and bullet points for clarity.\n"
    "Always be polite and helpful."
)
```

## 三、如何优化提示词（通用方法）

### 3.1 迭代式测试
- 准备一组测试用例（用户问题 + 期望答案风格）。
- 手动调整提示词，观察输出变化，记录哪些改动有效。
- 使用 A/B 测试对比不同提示词版本。

### 3.2 利用 LangChain 的 `FewShotPromptTemplate`
你可以创建一个包含示例的提示词模板，让模型学习输出风格。例如：

```python
from langchain.prompts import FewShotPromptTemplate, PromptTemplate

examples = [
    {"query": "推荐桂林适合摄影的景点", "answer": "桂林适合摄影的景点有：\n1. 相公山（日出）\n2. 漓江兴坪段（20元人民币背景）\n3. 龙脊梯田（金秋）"},
    {"query": "桂林3天2晚行程", "answer": "### 第1天\n上午：象鼻山...\n### 第2天\n..."}
]
example_prompt = PromptTemplate(input_variables=["query", "answer"], template="Q: {query}\nA: {answer}")
few_shot_prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    suffix="Q: {input}\nA:",
    input_variables=["input"]
)
```

### 3.3 结构化输出（Structured Output）
使用 LangChain 的 `with_structured_output` 强制 LLM 返回 JSON 或 Pydantic 模型，这在评分、路由等环节尤其有用。例如：

```python
from pydantic import BaseModel
from langchain.chat_models import init_chat_model

class Itinerary(BaseModel):
    days: list[dict]  # 每天行程
    tips: list[str]   # 特别提示

llm = init_chat_model(...).with_structured_output(Itinerary)
result = llm.invoke("桂林3天行程")
# result 现在是 Itinerary 对象，可直接使用
```

### 3.4 调整模型参数
- **温度（temperature）**：低温度（0~0.3）使输出更保守、格式稳定；高温度（0.7~1）增加创造性，适合内容生成。
- **top_p**：控制词汇多样性。
- **max_tokens**：限制输出长度，避免冗长。

### 3.5 添加负面约束
明确告诉模型“不要做什么”。例如：
> “不要使用‘根据我的知识’这类模糊表述，直接回答问题。不要重复用户的问题。”

## 四、总结与建议

- **内容丰富度**：通过优化生成提示词（角色设定、输出格式、示例、约束）可以显著提升。请将这部分提示词放在 Agent 的系统消息中或作为最终生成前的用户消息。
- **Agent Prompt 模板**：设计关键在于明确角色、工具使用规则、调用限制、错误处理。LangChain 的 `create_agent` 允许你传入 `system_prompt`，务必仔细打磨。
- **优化方法**：迭代测试、使用 few-shot、结构化输出、调整模型参数、添加负面约束。

在答辩中，你可以强调：**提示词是连接用户意图与模型能力的桥梁，我们通过反复实验和结构化设计，使系统能稳定输出高质量、可读性强的行程规划，这正是项目工程化的重要体现。**
